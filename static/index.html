RemoteDiag 사용 가이드

1. 에이전트 설치 및 실행

  1. 상단 헤더의 ⬇ agent.exe 버튼으로 에이전트를 다운로드합니다.
  2. 단말기가 연결된 Windows PC에서 agent.exe를 실행합니다.
  3. 콘솔에 서버 URL을 입력합니다. (예: wss://192.168.1.100:8443)
  4. 상단 에이전트 상태가 녹색으로 바뀌면 연결 성공입니다.

2. ADB 디바이스 연결

  1. 왼쪽 사이드바에서 ↺ 새로고침을 클릭합니다.
  2. 연결된 장치 목록에서 대상 장치를 클릭하여 선택합니다.
  3. 장치가 하나만 연결된 경우 자동으로 선택됩니다.

3. 디바이스 정보

장치를 선택한 후 섹션 헤더를 클릭하면 해당 정보를 읽어옵니다. 새로고침 버튼으로 재조회할 수 있습니다.

  • AT 정보: AT$$DBS, AT+CGDCONT? 결과 (AT 포트가 열려 있어야 함)
  • 네트워크: 네트워크 인터페이스 정보
  • DNS/DHCP 정보: IP 범위, DNS 서버, MTU 등 DHCP 설정 요약
  • 메모리 정보: 메모리 사용 현황
  • 프로세스 정보: 실행 중인 프로세스 목록

4. AT Command

  1. 왼쪽 사이드바에서 포트를 선택하고 Baudrate를 설정한 후 열기를 클릭합니다.
  2. AT Command 탭에서 명령어를 입력하고 전송을 클릭합니다.
  3. AT 접두사는 자동으로 추가됩니다. (+CSQ 입력 시 AT+CSQ로 전송)

5. ADB Shell

하단 입력창에 shell 명령어를 입력하고 실행 버튼 또는 Enter를 누릅니다.

↑ / ↓ 방향키로 명령 히스토리를 탐색할 수 있습니다.

6. Logs

  1. 소스 드롭다운에서 로그 파일을 선택합니다. (local0 / local1 / messages)
  2. ▶ 시작 버튼을 클릭하면 실시간 로그가 표시됩니다.
  3. ■ 정지 버튼으로 스트리밍을 중단합니다.
  4. ⬇ 저장 버튼으로 현재까지 수신된 로그를 파일로 저장합니다.

7. kmsg

  1. ▶ 시작 버튼을 클릭하면 커널 메시지(dmesg)를 표시합니다.
  2. 탭에 머무르는 동안 5초마다 자동 갱신됩니다.
  3. 다른 탭으로 이동하면 갱신이 중단되고, 다시 돌아오면 자동으로 재개됩니다.
  4. ⬇ kmsg 저장 버튼으로 현재 커널 메시지를 파일로 저장합니다.

8. 원격 제어

  1. 원격 제어 탭으로 이동합니다.
  2. ▶ 원격 제어 요청 버튼을 클릭합니다. 서비스가 실행 중이지 않으면 5초마다 자동 재요청합니다.
  3. 요청이 수락되면 원격 제어가 활성화되고, 관리자가 입력한 명령과 실행 결과가 이 화면에 표시됩니다.
  4. ■ 종료 버튼으로 세션을 종료합니다.
